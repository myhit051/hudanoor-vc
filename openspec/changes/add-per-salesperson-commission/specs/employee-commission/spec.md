## ADDED Requirements

### Requirement: คิดค่าคอมเฉพาะยอดขายของผู้ใช้ที่เลือก (Per-salesperson commission)
ระบบ SHALL ให้แต่ละแถวค่าคอม (`BranchCommission`) ระบุรายชื่อผู้ใช้ (`salespersonNames`) ได้หลายคน เพื่อจำกัดให้คิดค่าคอมจากเฉพาะยอดขายที่ผู้ใช้เหล่านั้นเป็นผู้บันทึก (`recorded_by`) เท่านั้น เมื่อไม่ระบุผู้ใช้ ระบบ SHALL คิดจากยอดขายรวมทุกคนในช่องทาง/สาขานั้นตามพฤติกรรมเดิม

#### Scenario: ไม่ระบุผู้ใช้ (ค่าเริ่มต้น)
- **WHEN** แถวค่าคอมมี `salespersonNames` ว่างหรือไม่กำหนด
- **THEN** ระบบคิดค่าคอมจากยอดขายรวมทุกผู้บันทึกในช่องทาง/สาขาที่ตรงกัน (เหมือนพฤติกรรมเดิม)

#### Scenario: เลือกผู้ใช้คนเดียว
- **WHEN** แถวค่าคอมมี `salespersonNames` = ชื่อ user 1 คน
- **THEN** ระบบรวมเฉพาะยอดขายที่ `recorded_by` ตรงกับ user คนนั้น แล้วคูณด้วยเรท

#### Scenario: เลือกผู้ใช้หลายคน
- **WHEN** แถวค่าคอมมี `salespersonNames` มากกว่า 1 คน
- **THEN** ระบบรวมยอดขายของทุก user ที่อยู่ในรายการที่เลือก แล้วคูณด้วยเรท

#### Scenario: การเทียบชื่อแบบ normalize
- **WHEN** ชื่อใน `salespersonNames` กับค่า `recorded_by` ต่างกันที่ตัวพิมพ์เล็ก/ใหญ่หรือช่องว่างหัวท้าย
- **THEN** ระบบเทียบแบบ normalize (เช่นเดียวกับการเทียบ channel/branch) และถือว่าตรงกัน

### Requirement: รายงานคอมและใบเงินเดือนใช้ตรรกะเดียวกัน
ระบบ SHALL ใช้เงื่อนไขการกรองตาม `salespersonNames` เดียวกันทั้งในรายงานค่าคอม (action `report`) และตอนสร้าง/สร้างใหม่ payroll run เพื่อให้ตัวเลขสอดคล้องกัน

#### Scenario: ตัวเลขรายงานตรงกับใบเงินเดือน
- **WHEN** มีแถวค่าคอมที่ระบุ `salespersonNames`
- **THEN** ยอดขายและค่าคอมที่แสดงในรายงานเท่ากับที่บันทึกในใบเงินเดือนของรอบเดียวกัน
